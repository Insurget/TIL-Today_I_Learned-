# C# 프로그램의 작동 원리

-----

## 1. 닷넷 프레임워크

- 2002년 MS에서 발표한 응용 프로그램 개발 환경으로서 프로세스 가상 머신에 속함.

- 네이티브언어 : 운영체제에서 곧바로 실행.

- 닷넷 프레임워크 기반 : 닷넷 프레임워크가 설치된 환경에서만 실행.

- CLR(Common Language Runtime) : 가상 머신 역할을 함. 프로세스(EXE)가 실행되면 메모리에 함께 적재됨.

- 닷넷프로그램은 외형상 EXE/DLL로 기존의 프로그램과 동일한 구조이지만 내부적으로는 CLR구성요소가 로드 되어 실행됨.

### C#입장에서의 닷넷프레임 워크

- C#컴파일러가 소스코드를 기계어가 아닌 IL(Intermediate Language)이라고 하는 중간 언어로 EXE/DLL 파일 내부에 생성함.

- 프로그램 시작시 CLR을 로드하는 코드를 자동으로 EXE 파일 내부에 추가함.

* 1. 컴파일러에 의해 삽입된 CLR 로더 코드가 실행되고,

* 2. CLR 로드 후 사용자의 중간 언어 코드를 실행함.
